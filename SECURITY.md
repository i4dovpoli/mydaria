# ⚠️ ВАЖЛИВО: Безпека токену

## Проблема

GitHub автоматично блокує push, якщо виявляє токени в коді. Це захист від витоку секретів.

## Рішення

### Варіант 1: Дозволити push (швидко, але менш безпечно)

1. Перейдіть за посиланням, яке GitHub надав у помилці
2. Дозвольте push
3. Спробуйте push знову

### Варіант 2: Зробити репозиторій приватним (рекомендовано)

1. Перейдіть на https://github.com/i4dovpoli/mydaria/settings
2. Прокрутіть вниз до "Danger Zone"
3. Натисніть "Change visibility" → "Make private"
4. Тепер токен буде безпечний

### Варіант 3: Використати GitHub Secrets (для GitHub Pages)

Якщо використовуєте GitHub Actions для деплою, можна використати Secrets.

## Поточний стан

Токен зберігається в `config.js`. Для публічного репозиторію це небезпечно, але для приватного - нормально.

## Рекомендація

Зробіть репозиторій приватним, якщо це особистий проект. Це найбезпечніше рішення.
