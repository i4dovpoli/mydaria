# Рішення для публічного репозиторію

GitHub автоматично видаляє токени з публічних репозиторіїв через Secret Scanning (захист від витоку секретів).

## Варіант 1: Зробити репозиторій приватним (РЕКОМЕНДОВАНО) ⭐

**Найпростіше рішення!** GitHub Pages працює і з приватним репозиторіями.

### Як зробити:

1. Перейдіть на https://github.com/i4dovpoli/mydaria/settings
2. Прокрутіть вниз до секції **"Danger Zone"**
3. Натисніть **"Change visibility"**
4. Виберіть **"Make private"**
5. Підтвердіть зміну

**Переваги:**
- ✅ Токен буде безпечний
- ✅ GitHub не буде його видаляти
- ✅ Сайт продовжить працювати на GitHub Pages
- ✅ Тільки ви та люди, яким ви надали доступ, зможуть бачити код

**Недоліки:**
- ⚠️ Репозиторій стане приватним (але сайт залишиться публічним!)

## Варіант 2: Використати GitHub OAuth App (складніше)

Якщо репозиторій має залишитися публічним, можна використати OAuth flow.

### Крок 1: Створити OAuth App

1. Перейдіть на https://github.com/settings/developers
2. Натисніть **"New OAuth App"**
3. Заповніть:
   - **Application name**: "MyDaria Gallery"
   - **Homepage URL**: `https://i4dovpoli.github.io/mydaria`
   - **Authorization callback URL**: `https://i4dovpoli.github.io/mydaria`
4. Натисніть **"Register application"**
5. Скопіюйте **Client ID**
6. Натисніть **"Generate a new client secret"** та скопіюйте **Client Secret**

### Крок 2: Оновити код

Потрібно буде змінити логіку авторизації - це складніше і потребує backend або GitHub Actions.

## Варіант 3: Використати GitHub Actions (середня складність)

Можна зберігати токен в GitHub Secrets і використовувати через Actions, але це також складніше.

## Рекомендація

**Найкраще рішення - зробити репозиторій приватним.** 

GitHub Pages працює з приватним репозиторіями, і ваш сайт залишиться публічним для всіх. Тільки код стане приватним.

### Після зміни на приватний:

1. Оновіть `config.js` з токеном
2. Зробіть commit та push
3. GitHub більше не буде видаляти токен
4. Все працюватиме як раніше!
