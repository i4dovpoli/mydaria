# Інструкція для тестування синхронізації фото

## Крок 1: Перевірка налаштувань

1. Відкрийте файл `config.js`
2. Переконайтеся, що токен встановлено (рядок 13)
3. Переконайтеся, що налаштовано репозиторій (рядки 24-26)

## Крок 2: Тестування на телефоні

1. Відкрийте сайт на телефоні (через GitHub Pages або локально)
2. Відкрийте консоль браузера (якщо можливо) або перевірте повідомлення
3. Натисніть "Додати фото"
4. Виберіть та завантажте фото
5. Подивіться на повідомлення:
   - ✅ Якщо все добре: "Успішно додано X фото! Фото додано до галереї, Збережено в GitHub Gist, Оновлено script.js в репозиторії"
   - ⚠️ Якщо помилка: перевірте консоль браузера (F12)

## Крок 3: Перевірка на ПК

1. Відкрийте сайт на ПК
2. **ВАЖЛИВО**: Натисніть Ctrl+F5 (або Cmd+Shift+R на Mac) для повного оновлення сторінки
3. Перевірте, чи з'явилися фото, додані з телефону
4. Відкрийте консоль браузера (F12 → Console)
5. Подивіться на повідомлення:
   - "✅ Завантажено актуальний BASE_IMAGES з GitHub: X фото"
   - "✅ Завантажено фото з Gist: X фото"

## Крок 4: Якщо фото не з'являються

1. Перевірте консоль браузера (F12) на наявність помилок
2. Перевірте, чи токен має права `repo`:
   - Перейдіть на https://github.com/settings/tokens
   - Знайдіть токен "darya"
   - Переконайтеся, що він має права "gist, repo"
3. Перевірте, чи токен правильний в `config.js`
4. Спробуйте очистити кеш браузера (Ctrl+Shift+Delete)

## Крок 5: Перевірка в GitHub

1. Перейдіть на https://github.com/i4dovpoli/mydaria
2. Відкрийте файл `script.js`
3. Знайдіть масив `BASE_IMAGES`
4. Перевірте, чи там є нові фото (якщо токен має права `repo`)

## Що має працювати:

✅ Фото зберігаються в GitHub Gist  
✅ Фото автоматично додаються в script.js (якщо токен має права `repo`)  
✅ Всі користувачі бачать нові фото при оновленні сторінки  
✅ Фото синхронізуються між всіма пристроями  

## Якщо щось не працює:

1. Перевірте консоль браузера (F12) - там будуть детальні повідомлення про помилки
2. Переконайтеся, що токен має обидва права: `gist` та `repo`
3. Переконайтеся, що репозиторій налаштовано правильно в `config.js`
